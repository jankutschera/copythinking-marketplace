# AI Engine Optimization (AEO)

Analyze and optimize content to be discoverable and citable by AI-powered search engines. Unlike traditional SEO, AEO focuses on becoming the source AI models cite when answering queries.

## Required Inputs

Always collect these five inputs before starting analysis:

1. **Content piece**: URL or paste the content to analyze
2. **Primary query**: What search query should this rank for?
3. **Target audience**: Who is searching for this?
4. **Content goal**: Inform, convert, educate, compare, etc.
5. **Format**: Blog post, product page, FAQ, guide, etc.

## Analysis Process

Follow this sequence for every AEO analysis:

### 1. Initial Assessment

Evaluate four dimensions:
- **Structure**: Headings, lists, Q&A pairs, table of contents
- **Clarity**: Answer prominence, directness, specificity
- **Authority**: Citations, credentials, dates, sources
- **Completeness**: Query coverage, topic depth, related questions

Provide: Overall score (X/10) + 2-3 sentence summary

### 2. Identify Strengths

## Core AEO Principles

### Answer First Pattern
Place direct answer in first 100 words. AI models heavily weight early content when extracting answers.

**Example:**
âœ… "X costs $50-200 depending on size. Basic models start at $50..."
âŒ "There are many factors to consider when pricing X. Let's explore..."

### Question Headings
Use H2/H3 as questions people actually ask:

**Examples:**
- âœ… "How much does X cost?"
- âœ… "What's the difference between X and Y?"
- âŒ "Pricing Information"
- âŒ "Comparison Details"

### Entity-Based Language
Use specific names, brands, and concepts:

**Examples:**
- âœ… "Mailchimp integrates with Shopify to sync customer data"
- âŒ "The tool works with e-commerce platforms"
- âœ… "According to Gartner's 2024 report"
- âŒ "According to industry research"

### Citation Signals
Include these authority markers:
- Author name and credentials
- Publication date
- Last updated date
- Source links for claims
- Specific data with attribution

### Conversational Patterns
Write how people speak, not how algorithms think:

**Examples:**
- âœ… "You can expect to pay $50-200 for installation"
- âŒ "Installation costs vary by configuration parameters"
- âœ… "This works best if you have 10+ employees"
- âŒ "Optimal for mid-sized organizational structures"

## Content Type Guidelines

### Product Pages
**Structure:** Quick overview â†’ What is it â†’ Features â†’ Pricing â†’ Who it's for â†’ Comparison â†’ FAQ

**Key elements:**
- Direct answer in first paragraph
- Feature benefits (not just features)
- Concrete pricing with tiers
- Use case scenarios
- Head-to-head comparison table
- 5-10 FAQ questions

### How-To Guides
**Structure:** Quick answer â†’ Prerequisites â†’ Step-by-step â†’ Troubleshooting â†’ Related guides

**Key elements:**
- TL;DR answer upfront
- Time/difficulty estimate
- Numbered steps with expected outcomes
- Common problems + solutions
- Links to related content

### Comparison Articles
**Structure:** Quick verdict â†’ At-a-glance table â†’ Detailed comparison â†’ Use cases â†’ Recommendation â†’ FAQ

**Key elements:**
- Winner declared immediately
- Comparison table in first 200 words
- Feature-by-feature analysis
- Real-world use case scenarios
- Clear final recommendation

### FAQ Pages
**Structure:** Grouped questions (Getting Started â†’ Features â†’ Troubleshooting â†’ Pricing)

**Key elements:**
- Questions as users ask them
- Direct 2-3 sentence answers
- No question left vague
- Links to detailed guides
- Update date visible

## Quality Verification

Before finalizing analysis, verify recommendations include:

**Structure checks:**
- [ ] TL;DR or direct answer in first 100 words
- [ ] Question-formatted headings
- [ ] Bulleted/numbered lists for scannability
- [ ] FAQ section with 5+ questions
- [ ] Table of contents if 800+ words

**Content checks:**
- [ ] Direct answers to primary query
- [ ] 3+ specific examples or data points
- [ ] Technical terms defined on first use
- [ ] Related questions addressed
- [ ] Comparison to alternatives (if relevant)

**Authority checks:**
- [ ] 2+ authoritative sources cited
- [ ] Author credentials visible
- [ ] Publication date shown
- [ ] Update date shown
- [ ] Original research or data included

**Semantic checks:**
- [ ] Topic cluster comprehensively covered
- [ ] Related subtopics addressed
- [ ] Entity-based language used
- [ ] Natural, conversational tone
- [ ] Context for AI understanding

## Common Optimization Mistakes

**Avoid these errors:**
- Burying answer below fold
- Using vague generic language
- Skipping concrete examples
- Ignoring related questions
- Forgetting source citations
- Including outdated information
- Over-optimizing for keywords
- Sacrificing human readability

## Skill Usage Notes

- Focus on authoritative content, not gaming systems
- Human value = AI citation potential
- Content quality always trumps optimization tricks
- Update this skill quarterly as AI search evolves
- Test recommendations on real content before declaring success
